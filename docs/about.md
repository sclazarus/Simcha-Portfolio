## What I do

I specialise in developer-facing documentation: API references, integration guides, onboarding tutorials, and release notes. I work in a **docs-as-code** workflow — Markdown in Git, pull requests for review, CI/CD for deployment. I'm comfortable reading diffs, filing bugs, and pairing with engineers to nail down the behaviour that matters.

Outside of professional work, I document my own technical projects — the [Projects](projects/index.md) section of this site. Writing while doing keeps my process honest and produces docs that reflect a real procedure, not an idealised one.
